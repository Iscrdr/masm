## 🗂️ 第 1 阶段：基础与工具（Day 1–7）

**目标**：能手写、编译、运行最简单的汇编程序。

* **Day 1**：安装工具（`nasm`, `ld`, `gdb`），了解 ELF 格式。练习：编译并运行 Hello World。
* **Day 2**：理解寄存器与内存模型（`rax`, `rbx`, `rcx`, `rdx`, `rsi`, `rdi`, `rsp`, `rbp` 等）。练习：编写 `add.asm`，实现两个数相加。
* **Day 3**：掌握基本指令：`mov`, `add`, `sub`, `mul`, `div`。练习：实现 `(3 + 5) * 2`。
* **Day 4**：条件跳转与标志寄存器：`cmp`, `je`, `jne`, `jg`, `jl`。练习：写一个比较程序判断正负。
* **Day 5**：循环与计数：用 `jmp` + `cmp` 实现 `for` 循环累加 1..10。
* **Day 6**：系统调用接口（Linux `syscall`）。练习：实现读取 stdin 并输出字符。
* **Day 7**：汇编调用 C 函数或从 C 调用汇编（链接外部函数）。练习：从汇编中调用 `printf` 或 `puts`。

---

## 🧠 第 2 阶段：函数调用与栈机制（Day 8–14）

**目标**：掌握栈帧结构与调用约定（System V ABI）。

* **Day 8**：理解 `call` / `ret` 的执行流程，保存/恢复寄存器。练习：手写 `sum(n)` 递归函数。
* **Day 9**：栈帧结构与栈指针操作（`rbp`, `rsp`）。练习：实现局部变量和 `push`/`pop` 操作。
* **Day 10**：System V ABI 参数传递规则（`rdi, rsi, rdx, rcx, r8, r9`）。练习：写 `add(a,b)` 并从 C 调用。
* **Day 11**：外部符号链接与调用 C 库函数。练习：在汇编中调用 `printf` 打印寄存器或内存值。
* **Day 12**：模块与全局变量（跨文件引用符号链接）。练习：分文件定义/引用全局变量。
* **Day 13**：练习递归实现 `factorial(n)`。
* **Day 14**：复盘并实现一个小型计算器的函数栈框架。

---

## ⚙️ 第 3 阶段：汇编层表达式求值（Day 15–20）

**目标**：理解从表达式到汇编的基本变换与实现技巧。

* **Day 15**：学习表达式树与中缀/后缀转换。练习：手工将 `1 + 2 * 3` 转换为后缀 `1 2 3 * +`。
* **Day 16**：实现后缀（逆波兰）求值算法（使用栈）。练习：用汇编实现后缀求值（支持 `+ - * /`）。
* **Day 17**：布尔运算与短路逻辑。练习：实现 `if a > b then 1 else 0` 的汇编生成与执行。
* **Day 18**：在汇编中设计简单的运行时类型模型（tag + data），用于区分 `int` / `bool` 等。
* **Day 19**：实现输出整数与布尔（`true`/`false`）的打印例程。
* **Day 20**：整合前述模块，完成 Mini Interpreter V1：输入表达式 → 求值 → 输出。

---

## 🧱 第 4 阶段：编译器后端思维（Day 21–27）

**目标**：将 AST 映射为汇编代码，处理寄存器分配与控制流生成。

* **Day 21**：设计 AST 节点（`NUM`, `ADD`, `MUL`, `IDENT` 等）并练习手动以树结构表示表达式。
* **Day 22**：临时变量管理与寄存器分配策略（简单的寄存器分配或基于栈的方案）。
* **Day 23**：处理表达式嵌套、递归和栈上保存中间值，避免溢出。
* **Day 24**：控制流（`if` / `while`）到汇编：生成标签与条件跳转。
* **Day 25**：实现生成 `.asm` 文件的流程，并用 `nasm + ld` 生成可执行文件。
* **Day 26**：实现简单优化（常量折叠、指令合并）。
* **Day 27**：实现一个简易 codegen：输入 AST → 输出汇编字符串并测试。

## 🚀 第 5 阶段：项目实战（Day 28–30）

* **Day 28**：定义你语言的最小语法（例如：`let`, `print`, `expr`）。写出 BNF 或 EBNF 草案。
* **Day 29**：实现一个 Mini Compiler：词法分析 → 语法分析 → 生成汇编（`.asm`）。
* **Day 30**：端到端测试：`source.luna` → `generated.asm` → 链接 → `./out` 并验证输出。

**最终产物示例流程**：

```
input.luna  →  luna.asm  →  nasm/ld  →  luna.out  →  运行
```

## 🔧 每日练习建议

* 每日安排：1 小时理论学习 + 1–2 小时动手编写汇编与测试。
* 每天用 `objdump -d` 或 `ndisasm` 反汇编你编译的目标文件，观察指令流。
* 多用 `gdb` 单步调试（查看寄存器与内存），尤其在处理栈帧与调用时。
