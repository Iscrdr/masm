## MOV 、ADD、sub、Mul、DIV常用的几条汇编指令

### ✅ 1. mov 功能：把数据从一个位置移动到另一个位置，不做计算，不改变标志位，只是“复制”。

用途：
- 把立即数放入寄存器
- 寄存器之间传值
- 内存与寄存器之间读写
实例：
```
mov r0,5      ; 把5传入r0,r0 = 5
mov r1,r0     ; 把r0的值传入到r1,r1 =r0 = 5
mov r2,[addr] ; 把内存addr的值读入r2
mov [addr],r1 ; 把r1的值写入内存addr
```
概念非常简单：搬运，不计算。

### ✅ 2. add —— 加法,会影响标志位（ZF, CF 等），用于判断结果是否为 0、是否溢出等。
功能：r0 = r0 + r1 或 r0 = r1 + imm 等。
示例：
```
add r0, r0, #1      ; r0 += 1
add r2, r1, r0      ; r2 = r1 + r0
```
典型用途：
- 计数器 ++
- 栈指针修改
- 指针偏移（地址计算）

### ✅ 3. sub —— 减法,也会影响标志位。
功能：r0 = r0 - r1。
示例：
```
sub r0, r0, #1      ; r0 -= 1
sub r1, r2, r3      ; r1 = r2 - r3
```
用途：循环计数，自减，比较（很多架构里 cmp a,b 等价于 sub a,b 但不写结果）。

### ✅ 4. mul —— 乘法
各架构略微不同，但核心相同：寄存器 × 寄存器 → 结果（可能更大）
简单示例：
```
mul r0, r1, r2      ; r0 = r1 * r2
```
有时结果需要多个寄存器保存（如 x86 32→64 位扩展）。
用途：数值计算、标量运算、地址偏移（例如计算数组索引 val[i * 4]）
### ✅ 5. div —— 除法
除法通常比加法慢得多，而且指令复杂。
不同架构差异最大，但基本思想一致：
```
div r1, r0, r2      ; r1 = r0 / r2
```
部分平台还要额外寄存器保存余数。
用途：极少在性能关键代码中直接使用，通常编译器会优化成乘法与移位。
