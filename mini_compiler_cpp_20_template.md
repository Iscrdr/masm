# C++20 Compiler Project Template (LLVM-based, Analysis-focused)

This is a minimal, production-oriented starting template for a compiler designed for:

- Field-sensitive dependency analysis
- Escape analysis
- Function summary propagation
- Static memory elimination + ref-count fallback
- LLVM backend codegen

---

# 📁 Project Structure

This compiler supports **.ppl source files** (example: `User.ppl`, similar to `User.java` or `hello.c`).

The frontend is designed around file-based module compilation:

- Each `.ppl` file = one compilation unit
- Supports multi-file project linking later
- Entry file can be specified (e.g. main.ppl)

---

```
User.ppl
Main.ppl
Math.ppl
```

# ⚙️ CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.20)
project(MiniCompiler CXX)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_EXPORT_COMPILE_COMMANDS ON)

# LLVM config (adjust if needed)
find_package(LLVM REQUIRED CONFIG)

message(STATUS "Found LLVM ${LLVM_PACKAGE_VERSION}")
message(STATUS "Using LLVMConfig.cmake in: ${LLVM_DIR}")

include_directories(${LLVM_INCLUDE_DIRS})
add_definitions(${LLVM_DEFINITIONS})

file(GLOB_RECURSE SOURCES
    src/*.cpp
)

add_executable(minicompiler ${SOURCES})

llvm_map_components_to_libnames(llvm_libs
    core
    support
    irreader
    analysis
    target
    mc
    x86codegen
)

target_link_libraries(minicompiler ${llvm_libs})
```

# 🧠 Core Design (Important IR Rule)

All IR must be:

- index-based (NO raw pointer ownership)
- arena allocated
- SSA-like
- explicit field access

---

# 🧩 Example MIR (design target)

```
v1 = param User
v2 = field v1, "id"
v3 = add v2, 1
return v3
```
