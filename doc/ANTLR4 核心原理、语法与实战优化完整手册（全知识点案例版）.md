# 一、ANTLR4 的工作原理（含全套实战案例）

ANTLR4 是自适应LL\(\*\) 解析器生成器，核心革新：无回溯、支持直接左递归、动态语法预判，兼容所有上下文无关文法。本章所有原理均搭配可直接运行的极简案例。

## 1、ANTLR4 的基本工作原理（含案例）

**核心原理**：ANTLR4 是文法驱动的代码生成工具，用户编写 `.g4` 文法文件，工具自动生成 Lexer、Parser、语法树、监听器全套代码，实现文本自动解析，无需手写词法语法逻辑。

**ANTLR4 核心革新**：摒弃旧版本 LL\(k\) 固定预测与回溯机制，采用自适应LL\(\*\) 动态预判，无需人工消除左递归。

**实战极简案例**（自定义数字表达式文法）

编写 `Expr.g4` 文法文件：

```antlr4
grammar Expr;
prog: expr+ EOF;
expr: expr '+' INT | INT; // ANTLR4 原生支持左递归（ANTLR3 必须手动消除）
INT: [0-9]+;
WS: [ \t\n]+ -> skip;

```

执行 ANTLR4 代码生成后，自动产出：`ExprLexer`、`ExprParser`、`ExprListener`、语法树节点类，可直接解析 `12+34+56` 表达式文本。

## 2、ANTLR4 的解析流程（含案例）

**完整四阶段流程**：词法扫描 → 语法解析 → 语法树生成 → 自定义后续处理，全程无回溯线性执行。

**流程落地案例**（解析文本：`10+20`）

1、词法扫描：切割 Token 流 → `INT(10)、+、INT(20)`，过滤空格；

2、语法解析：匹配 `expr -> expr + INT` 规则，校验语法合法；

3、树结构生成：生成完整表达式语法分析树；

4、后续处理：通过 Listener 遍历树，计算表达式结果为 30。

## 3、ANTLR4 的语法规则定义（含案例）

**核心规则规范**：词法规则大写开头、语法规则小写开头，严格分离，ANTLR4 原生支持左递归。

**对比案例（新旧版本差异）**

ANTLR4 合法写法（直接左递归，极简）：

```antlr4
expr: expr '+' INT | INT;

```

ANTLR3 旧版本必须改写为无左递归形式，繁琐且可读性差：

```antlr4
expr: INT exprTail?;
exprTail: '+' INT exprTail?;
```

## 4、ANTLR4 的抽象语法树生成（含案例）

**原理**：ANTLR4 默认生成完整语法分析树，可通过 API 过滤冗余节点，二次加工生成精简 AST。

**案例对比**：解析 `1+2+3`

语法分析树（原生自带）：完整保留所有规则节点、Token 节点，层级冗余；

AST（二次优化后）：剔除多余嵌套节点，仅保留「根表达式、三个数字节点、两个加号节点」核心语义结构，可直接用于计算求值。

## 5、ANTLR4 的语义动作与监听器（含案例）

**核心机制**：ANTLR4 放弃内嵌代码，采用 Listener/Visitor 无侵入解耦设计，文法可跨语言复用。

**Listener 实战案例（表达式求值）**

遍历语法树，进入/离开节点自动回调，实现表达式计算：

```java
// 重写退出表达式节点方法，递归计算结果
@Override
public void exitExpr(ExprParser.ExprContext ctx) {
    if (ctx.INT() != null) {
        // 单个数字
        ctx.value = Integer.parseInt(ctx.INT().getText());
    } else {
        // 左表达式 + 右数字
        ctx.value = ctx.expr().value + Integer.parseInt(ctx.INT().getText());
    }
}

```

## 6、ANTLR4 的错误处理与恢复机制（含案例）

**原理**：默认开启自适应容错，报错不中断解析，自动修复简单语法错误。

**容错案例**：解析错误表达式 `10 + * 20`

ANTLR4 自动识别多余 `*` Token，删除异常 Token，继续解析后续 20，完成整体语法树构建，同时抛出精准语法错误，不会程序崩溃。

# 二、ANTLR4 语法规则与语法描述（含全套实战案例）

## 1、ANTLR4 语法规则定义（案例）

规则分为词法（大写）、语法（小写），标准格式：`规则名: 匹配体;`

完整定义案例（简易变量语法）：

```antlr4
stmt: varDef;       // 语法规则：语句为变量定义
varDef: VAR ID EQ INT SEM;
VAR: 'var';         // 词法规则：关键字
ID: [a-z]+;
EQ: '=';
INT: [0-9]+;
SEM: ';';
WS: [ ]+ -> skip;
```

## 2、ANTLR4 语法描述语言（案例）

核心语法元素实战演示：分支、可选、多次、分组

```antlr4
// 支持加减表达式，可选末尾空格
calc: expr WS? EOF;
expr: expr ('+'|'-') INT | INT; // 分支匹配加减

```

## 3、ANTLR4 语法规则组成（案例）

以 `expr: expr '+' INT | INT;` 为例拆解完整组成：

规则名：expr（语法规则小写）；匹配体：分支表达式；无修饰符、无标签、分号结尾，为标准完整规则。

## 4、ANTLR4 语法规则顺序（核心案例）

**关键特性**：词法规则先定义优先级更高，解决关键字冲突。

正确案例（无冲突）：

```antlr4
VAR: 'var'; // 先定义关键字
ID: [a-zA-Z]+; // 后定义标识符

```

错误案例（关键字被识别为标识符）：先ID后VAR，`var` 会被识别为普通变量名。

## 5、ANTLR4 语法规则关联（案例）

嵌套\+递归关联案例：

```antlr4
prog: stmt+;     // 顶层引用子规则
stmt: exprStmt;
exprStmt: expr SEM;
expr: expr '+' INT | INT; // 自递归关联

```

## 6、ANTLR4 语法规则引用（跨文件案例）

大型语法拆分复用案例：

1、新建 `BaseToken.g4` 定义通用 Token（INT、ID、符号）；

# 四、ANTLR4 错误处理与恢复机制（含全套案例）

## 1、ANTLR4 错误处理概述（案例场景）

## 2、词法和语法错误（区分案例）

**词法错误**：输入 `12#3`，\#为非法字符，无法生成Token；

**语法错误**：输入 `10 + * 20`，Token 合法但序列不符合语法规则。

## 3、错误恢复策略（三大策略案例）

1、Token插入修复：`10 + 20` 漏写数字 `10 +`，自动补默认0，完成解析；

2、Token删除修复：`10 + * 20`，自动删除非法\*；

3、同步恢复修复：多行代码缺失分号，自动同步换行，继续解析下一行。

## 4、错误消息定制（中文报错案例）

重写错误监听器，将默认英文报错改为：`第1行第4列：语法错误，预期数字，当前符号为*`，适配国内开发者使用。

## 5、错误处理示例（综合实战）

错误代码：

```Plain Text

```

ANTLR4 容错效果：补全第一行末尾分号、删除第二行末尾\*，完整解析两段代码，同时输出精准报错信息。

## 6、错误处理最佳实践（落地案例）

生产实践：自定义全局错误监听器、开启批量容错、核心语法关闭容错、收集错误日志，适配 IDE 语法检测工具。
