# 类名（Class Name）在编译器中的识别与处理

## 1. 类名基础概念

### 1.1 类名的定义
- 类名是什么

类名是程序员位一个类class指定的标识符，例如：
```
class Student
{
};
```
这里 Student就是一个类名，编译器会把它当做一个”用户自定义的类型User Defined Type“处理

- 类名与标识符的关系
标识符Identifier是程序中的名称，例如：
```
Student
age
printf
main
```
都属于标识符，类名本质上也是一种特殊的标识符

    变量名 age
    函数名 printf()
    类名  Student
    命名空间 std
- 类名与对象的关系
```
类
class Student{};
对象
Student s;
```
类->模板
对象->实例

- 类名与类型（Type）的关系

    对于编译器来说，int age, int 是类型，Student s ; Student 是类型
    因此：类型=用户自定义类型名称   

### 1.2 类名的作用
- 表示用户自定义类型

        编译器通过类名区分不同的类型
- 支持面向对象程序设计
        类名能够描述现实对象

- 实现代码组织与模块化
        类名帮助构建代码结构
- 提高代码可读性与可维护性


### 1.3 类名与面向对象特性
- 封装（Encapsulation）
        类名代表一个封装单元
- 继承（Inheritance）

- 多态（Polymorphism）
        运行时决定调用哪个虚函数
- 抽象（Abstraction）
        只暴露接口，隐藏实现

## 2.1 标识符与关键字识别
- 编译器如何区分关键字与标识符
```
class Student {};
```
词法分析器扫描字符流，产生Token:
```
class      -> KEYWORD
Student    -> IDENTIFIER
{          -> LBRACE
}          -> RBRACE
;          -> SEMICOLON
```
此时，Student还不知道是不是类名，知识标识符，编译器通过查找关键字表：
```
class
if
while
return
```
class 是关键字，Student不在关键字表中，那么就是标识符IDENTIFIER

# 3. 语法分析阶段中的类名处理

## 3.1 类声明解析
- 类定义语法结构
```
class Student
{
}
```
语法分析器根据文法：
```
ClassDeclaration
    -> class Identifier { }
```
识别出 Student 是类名
- 类声明与类定义

- 前向声明
class Student; 
告诉编译器 ：Student类型存在

## 3.2 抽象语法树（AST）中的类名
- 类节点表示
类本身对应一个专门的节点名称
```
class Student {
}
```
对应的AST：
```
ClassDeclaration
 └── Name: Student
```
常见的节点：
ClassDeclaration

    • ClassDef
    • ClassNode
    • TypeDeclaration
包含信息：
| 属性   | 说明               |
| ---- | ---------------- |
| 类名   | Student          |
| 修饰符  | public、abstract等 |
| 泛型参数 | `<T>`            |
| 继承关系 | extends          |
| 实现接口 | implements       |
| 成员列表 | 字段、方法、内部类        |


- 类成员节点关联
```
class Student {
    String name;
    int age;

    void study() {}
}
```
AST结构：
```
ClassDeclaration(Student)
├── FieldDeclaration(name)
├── FieldDeclaration(age)
└── MethodDeclaration(study)
```
字段（Field）是类节点的子节点
方法（Method）是类节点的子节点
内部类也是类节点的子节点
关系表示为：
```
Class
 ├── Member
 ├── Member
 └── Member
```
类节点通过子节点关联其成员节点

- 继承关系节点表示
```
class Student extends Person {
}
```
AST
```
ClassDeclaration(Student)
 └── ExtendsClause
      └── TypeReference(Person)
```
或者
```
ClassDeclaration
├── Name(Student)
└── SuperClass(Person)
```
复杂的情况
```
class Student extends Person
              implements Runnable, Serializable {
}
```
AST
```
ClassDeclaration(Student)
├── Extends
│    └── Person
└── Implements
     ├── Runnable
     └── Serializable
```
因此：

        ExtendsClause
        • BaseClass
        • SuperClass
        • TypeReference
        等节点用于表示继承关系
## 3.3 类名与语法规则
- 类定义规则
以java为例：
```
classDeclaration
    : 'class' Identifier classBody
    ;
```
代码：
```
class Student {
}
```
解析过程：
```
class
 ↓
Student
 ↓
{
}
```
生成AST：
```
ClassDeclaration
 └── Identifier(Student)
```
Student是类名
来自语法规则中的Identifier
- 类引用规则
类名不仅出现在定义处，也会出现在引用处。
```
Student s;
```
AST：
```
VariableDeclaration
├── TypeReference(Student)
└── Variable(s)
```
这里的Student不是类定义的节点，而是TypeReference或者Identifier节点，区别：
```
| 场景               | AST节点            |
| ---------------- | ---------------- |
| class Student {} | ClassDeclaration |
| Student s;       | TypeReference    |
| new Student()    | TypeReference    |
| Student.method() | NameReference    |

```
- 类型声明规则
```
List<Student> students;
```
AST:
```
VariableDeclaration 
└── GenericType
     ├── List
     └── Student
```
或者
```
TypeDeclaration
├── BaseType(List)
└── TypeArgument(Student)
```
    List 是类型名
    • Student 是类型参数
    • 都属于类型声明的一部分
再如：Student[] arr;
AST：
```
ArrayType
 └── TypeReference(Student)
```

综合示例：
```
class Student extends Person {

    String name;

    void study() {
    }
}
```
AST:
```
ClassDeclaration(Student)
├── SuperClass(Person)
├── FieldDeclaration
│     ├── Type(String)
│     └── Name(name)
└── MethodDeclaration(study)
```
| 概念   | AST节点类型                                | 示例               |
| ---- | -------------------------------------- | ---------------- |
| 类定义  | `ClassDeclaration`                     | `class Student`  |
| 类成员  | `FieldDeclaration`、`MethodDeclaration` | `name`、`study()` |
| 继承关系 | `ExtendsClause`、`SuperClass`           | `extends Person` |
| 类引用  | `TypeReference`                        | `Student s`      |
| 类型声明 | `TypeDeclaration`、`GenericType`        | `List<Student>`  |

# 4. 语义分析阶段中的类名处理

## 4.1 符号表管理
符号表是编译器用于存储剩下中各种标识信息的数据结构
```
class Student {}
class Teacher {}
```
编译器可能建立：
| 名称      | 类型    | 信息  |
| ------- | ----- | --- |
| Student | Class | 类定义 |
| Teacher | Class | 类定义 |

- 类名的注册过程
```
class Student {
}
```
会执行：
```
注册类名 Student
↓
创建符号记录
↓
加入全局符号表
```
示例：
```
Global Symbol Table
Student → ClassSymbol
Teacher → ClassSymbol
```
伪代码：
```
visitClassDeclaration(node):
    symbol = new ClassSymbol(node.name)
    symbolTable.insert(node.name, symbol)
```
- 类型信息存储
类名不仅存储名字，还要存储完整的类型信息
```
class Student {
    int age;
    String name;
}
```
符号记录：
```
ClassSymbol(Student)

name = Student

fields:
    age  : int
    name : String

methods:
    ...
```
这样后面进行成员访问检查时：
```
Student s;
s.age;
```
- 符号表查找机制
当出现Student s; 编译器需要查找 Student 是否存在，过程：当前作用域->父作用域->全局作用域
```
Local Scope
    ↓
Package Scope
    ↓
Global Scope
```
找到后返回，时间复杂度通常O(1)

## 4.2 类名解析（Name Resolution）
- 作用域查找
```
class Student {}

void test() {
    Student s;
}
```
编译器查找顺序
```
当前作用域
    ↓
外层作用域
    ↓
全局作用域
```
如果存在嵌套，按照作用域链查找：
```
class A {
    class Student {}
}
```
Student s; 可能对应的是 A.Student，而不是全局的 Student
- 命名空间查找
很多语言支持命名空间。例如 C++：
```
namespace School {
    class Student {};
}

namespace Company {
    class Student {};
}
```
使用时：
```
School::Student s1;
Company::Student s2;
```
解析过程：
```
School
 ↓
Student
 ↓
School::Student
```
- 类型匹配
Student s ; 编译器首先要确定Student是一个类类型，
```
int Student;
Student s;
```
Student is not a type

## 4.3 语义检查
名称解析成功后，还需要进行各种合法性检查
- 重复定义检查
```
class Student {}
class Student {}
```
编译器注册第二个类时：
```
Student
↓
已存在
↓
错误
```
duplicate class definition
```
if(symbolTable.contains(name))
    error("duplicate definition");
```
- 未定义类型检查
```
Teacher t;
```
但是不存在：
```
class Teacher {}
```
查找过程
```
Teacher
 ↓
未找到
 ↓
错误
```
cannot find symbol Teacher
- 访问权限检查
```
class Student {
    private int age;
}
```
外部访问
```
Student s;
s.age;
```
语义检查分析：
```
age
↓
private
↓
当前类外访问
↓
非法
```
- 类型一致性检查
```
Student s;
Teacher t;

s = t;
```
过程：
```
左侧类型 Student
右侧类型 Teacher
↓
是否兼容？
↓
否
```
incompatible types

## 4.4 错误处理与诊断
- 类名解析失败
```
Studnet s;
Student s;
```
编译器查找：Student -> 不存在，报错
很多编译器还会相似度匹配：
error: unknown type 'Studnet'
did you mean 'Student'?
- 类型冲突错误
```
class Student {};

int Student;
```
同一作用域：
```
Student
↓
Class
Student
↓
Variable
```
发生名字冲突。
- 编译器错误提示机制
编译器会提供提示机制：
```
Studentt s;
```
输出：
```
Line 12, Column 5
Unknown type Studentt
```
最终目标是把源码中的类名从一个简单的字符串（如 "Student"）解析成编译器内部的类型对象（如 ClassSymbol(Student)），并保证所有引用都合法且类型正确
```
类定义出现
    ↓
注册到符号表
    ↓
保存类型信息
    ↓
名称解析(Name Resolution)
    ↓
作用域查找
    ↓
命名空间查找
    ↓
获得ClassSymbol
    ↓
语义检查
      ├─ 重复定义
      ├─ 未定义类型
      ├─ 访问权限
      └─ 类型一致性
    ↓
错误诊断与提示
```
