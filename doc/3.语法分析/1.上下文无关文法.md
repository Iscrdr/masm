## 一、上下文无关文法简介
1、上下文无关文法定义

    上下文无关文法是描述程序语言语法结构的重要工具。一个CFG表示为：

            G=(Vn​,Vt​,P,S)
其中：
        
        Vn是非终结符集合
        vt是终结符符集合
        P是产生式集合
        S是开始符号
示例文法：

        E -> E + T | T
        T -> T * F | F
        F -> (E) | id
其中

            非终结符：E,T,F
            终结符：+，*，（，）id
            开始符号：E

2、产生式和推导

    产生式形式：A -> a，表示非终结符A可以替换成串a
    例如：E -> E * T
    推导：从开始符号不断应用产生式生成句子的过程
例如：

        E
        ⇒ E + T
        ⇒ T + T
        ⇒ F + T
        ⇒ id + T
        ⇒ id + F
        ⇒ id + id
左推导：每次替换最左边的非终结符

        E 
        ⇒ E+T
        ⇒ T+T
        ⇒ F+T
        ⇒ id+T
        ⇒ id+id
右推导：每次替换右边的非终结符

        E 
        ⇒ E+T
        ⇒ E+F
        ⇒ E+id
        ⇒ T+id
        ⇒ id+id

3、左递归和左因子消除

    左递归：
    若存在 A -> Aa ，则称文法存在左递归
    例如：E → E + T | T
    这里 E -> E + T,就是左递归。
    为什么要消除，因为递归下降分析会无限循环
        E(){
            E(); //死循环
        }
    消除直接左递归，
    形式：
        A → Aα | β
    变换为：
        A → βA'
        A'→ αA' | ε
    示例：
        E → E + T | T
    变换为：
        E → T E'
        E'→ + T E' | ε
    左因子提取：当多个产生式拥有公共前缀：
        A → αβ1 | αβ2
    改写为：
        A → αA'
        A'→ β1 | β2
    示例，原文法：
        S → if E then S
         | if E then S else S
    公共前缀：if E then S
    改写为：
        S → if E then S S'
        S'→ else S | ε

3、FIRST集和FOLLOW集
First集定义：从X 推导出的串的第一个终结符集合，计算规则

情况1：如果x是终结符，则FIRST(x) = {x}

情况2：A→ε 则 ε ∈ FIRST(A)

情况3：A→X1X2…Xn 加入 FIRST(X1)-{ε}，若 ε ∈ FIRST(X1) 继续看FIRST(X2)

示例：
    E → T E'
    E'→ + T E' | ε

    T → F T'
    T'→ * F T' | ε

    F → (E) | id

求：FIRST(F) 
    F → (E)
    F → id
所以：FIRST(F)={(,id}
    FIRST(T')
     T'→ *FT' | ε
所以：FIRST(T')

4、FOLLOW集
FoLLOW（A)：可能紧跟A后面的终结符集合

规则1

    开符号：$ ∈ FOLLOW(S)
规则2

    若
        A → αBβ 
    则
        FIRST(β)-{ε}
        加入FOLLOW(B)
规则3

    若

        A→αB 、
        或者 
        A→αBβ且ε∈FIRST(β)
    则
        FOLLOW(A)
        加入FOLLOW(B)
示例：

    E → T E'
    E'→ +TE' | ε
    T → FT'
    T'→ *FT' | ε
    F → (E) | id
结果：

    FOLLOW(E)={$,)}
    FOLLOW(E')={$,)}
    FOLLOW(T)={+,),$}
    FOLLOW(T')={+,),$}
    FOLLOW(F)={*,+,),$}


7、LL(1) 文法和语法分析
定义
    满足:

    对任意： A→α | β
    必须：
    条件1 ： FIRST(α)∩FIRST(β)=∅
    条件2：若 ε∈FIRST(α)，则 FIRST(β)∩FOLLOW(A)=∅

    特点
    从左到右扫描（L）
    最左推导（L）
    向前看一个符号（1）

8、LL(1)预测分析表构建

步骤：
    对于 A→α
    若 a∈FIRST(α) 填 

        | A | a   |
        | - | --- |
        |   | A→α |
    若 ε∈FIRST(α) 则：
    对于 b∈FOLLOW(A) 填 M[A,b]=A→α
示例：
    
    E → TE'
    E'→ +TE' | ε
    T → FT'
    T'→ *FT' | ε
    F → (E) | id
预测分析表：

    |    | id  | + | *    | (    | )   | $ |   |   |
    | -- | --- | - | ---- | ---- | --- | - | - | - |
    | E  | TE' |   |      |      | TE' |   |   |   |
    | E' |     |   | +TE' |      |     | ε | ε |   |
    | T  | FT' |   |      |      | FT' |   |   |   |
    | T' |     |   | ε    | *FT' |     |   | ε | ε |
    | F  | id  |   |      |      | (E) |   |   |   |

## 三、自底向上的语法分析
1、自底向上语法分析概述

    从输入串开始：
        id + id * id
    不断规约：
        id→F
        F→T
        T→E
    最终 E，即：最右推导逆过程

    2、LR（1）文法特性
LR含义 ：

    L：从左到右扫描
    R：最右推导逆过程
    1：向前看一个符号
LR(1)项：

    形式：
        [A→α·β,a]
    例如：[E→E·+T,$]
    表示：
        已经识别 E，期待+T，展望 $
LR(0)项：
    没有展望符：A→α·β

LR(1)项集构建：

    闭包 Closure(I)：
        若：[A→α·Bβ,a]
        则，对 B→γ，加入 [B→·γ,b] 其中 b ∈ FIRST(βa)，直到不再增加
    GOTO(I,X)：
        若：A→α·Xβ
        则：A→αX·β
        形成新状态

示例，文法：

    S'→S
    S→CC
    C→cC
    C→d
初态：

    I0:
    S'→·S,$
    S→·CC,$
    C→·cC,c/d
    C→·d,c/d
读入 c ，得到：

    I1:
    C→c·C
    C→·cC
    C→·d
LR(1)分析表
分
ACTION表:

        | 状态 | 终结符    |
        | -- | ------ |
        | 0  | shift3 |

动作：
    shift
    reduce
    accept

GOTO表:

    | 状态 | 非终结符 |
    | -- | ---- |
    | 0  | 2    |

LR分析过程
维护：
    状态栈：0
    符号栈：$
输入：id+id$
循环：

    查ACTION
    移进
    归约
    接受

示例：
    
    状态：0
    输入：id+id$
    动作：shift 5
    变 0 5
    规约 F -> id
    弹栈：1个符号
    压入：F

LR(0)、SLR(1)、LALR(1)、LR(1)比较：

    | 文法类型    | 能力 | 状态数 |
    | ------- | -- | --- |
    | LR(0)   | 最弱 | 最少  |
    | SLR(1)  | 较弱 | 少   |
    | LALR(1) | 较强 | 中   |
    | LR(1)   | 最强 | 最多  |
关系：

    LR(0)
    ⊂ SLR(1)
    ⊂ LALR(1)
    ⊂ LR(1)

## 六、文法规范化
- Chomsky范式（CNF）

    形式：
        A→BC
        A→a
    例如：
        S→AB
        A→a
        B→b
- Greibach范式（GNF）

    形式：
        A→aα
    即：右部必须以终结符开头
    例如：A→aBC
## 7、等价转换
两个文法，若生成的语言相同：L(G1​)=L(G2​)，则称等价，常见的转换：

        消除左递归
        提取左因子
        消除空产生式
        消除单产生式
        消除无用符号
总结：
LL(1)分析流程：

    文法
    ↓
    消除左递归
    ↓
    提取左因子
    ↓
    求FIRST/FOLLOW
    ↓
    构造预测分析表
    ↓
    预测分析
LR(1)分析流程：

    增广文法
    ↓
    构造LR(1)项集簇
    ↓
    构造ACTION/GOTO表
    ↓
    移进-归约分析
    ↓
    接受或报错
