# 消除左递归

## 1. 什么是左递归？

如果一个非终结符在推导时，最左边又出现它自己，就称为**左递归**。

例如：
    A → Aα ∣ β
其中：
- `Aα`：产生式右边最左边仍然是 `A`，属于左递归；
- `α` 表示任意符号串；
- `β` 表示不以 `A` 开头的产生式。
---
## 2. 为什么要消除左递归？
考虑文法：
    E → E + T ∣ T

如果使用递归下降分析，分析 `E` 时：

```text
分析 E
→ 先使用 E → E + T
→ 又要分析 E
    → 再使用 E → E + T
        → 又分析 E
            ...
```
程序会不断调用自己：

```c
void E(){
    E();   // 无限递归
}
```
因此必须改写文法。

---
## 3. 消除左递归的方法
对于：
    A → Aα ∣ β
变换为：

    A  → βA′
    A′ → αA′ ∣ ε

其中：

- 原来递归部分 `Aα` 提取出来放到新符号 `A'` 中；
- `A'` 表示“后面可以继续重复出现的部分”。

---

## 4. 具体示例
原文法：
    E → E + T ∣ T
这里：
- `A = E`
- `α = +T`
- `β = T`

按照公式：

E → TE′ 
E′ → + TE′ ∣ ε

---

## 5. 推导一个字符串看看

假设要推导：

```text
id + id + id
```

### 原文法的推导

 E ⇒ E+T ⇒ E+T+T ⇒ T+T+T

存在左递归。

### 新文法推导
文法：
    E → TE′
    E′ → +TE′∣ε

推导：E → TE′
令第一个 `T => id`，
得到：id E'
继续使用：E' => +TE'
得到：id + T E'
令：T => id
得到：id + id E'
再次使用：E' => +TE'
得到：id + id + T E'
令：T => id
得到：id + id + id E'
最后：E′ ⇒ ε
于是：id + id + id

成功完成推导。

---

## 6. 图示理解

原文法：

```text
E
├── E
│   ├── E
│   │   └── T
│   └── + T
└── + T
```
不断向左展开。
消除后：
```text
E
├── T
└── E'
    ├── + T
    └── E'
         ├── + T
         └── ε
```

变成了向右扩展，不会无限递归。

---

## 7. 再看一个例子

原文法：
A → Aa ∣ Ab ∣ c
左递归部分：
Aa,Ab

非左递归部分：c
消除左递归：

A -> cA'

A′ → aA′∣ bA′ ∣ ε

例如推导字符串：

```text
caba
```

推导过程：
A
⇒ cA'
⇒ caA'
⇒ cabA'
⇒ cabaA'
⇒ caba


---

## 总结

左递归：

A → Aα1 ​∣ Aα2 ​∣⋯∣ β1​ ∣β 2​

消除后：

A → β1​A′ ∣ β2​A′
A′→ α1​A′ ∣ α2​A′ ∣ ε

**核心思想：**

> 把“无限向左递归”改成“先产生一个基本部分，再不断向右追加重复部分”。
