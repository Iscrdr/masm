FIRST 集定义
    
    对于一个文法符号串 α，FIRST(α) 表示：
    从 α 开始推导时，最终得到的句型中，第一个可能出现的终结符的集合。

现在来看：F→ (E) ∣ id
    F 有两个候选式：
    
第一个候选式 F→(E)
    
    右边第一个符号就是终结符：(

因此：

    FIRST((E))={(}
第二个候选式    F→id

    右边第一个符号就是：id

所以：FIRST(id)={id}

由于 F 可以选择这两个产生式中的任意一个，因此：

    FIRST(F)=FIRST((E))∪FIRST(id)

即：

    FIRST(F)={(}∪{id}

所以：

    FIRST(F)={(, id}
	​

为什么没有 E？很多同学会疑惑：

既然有 F → (E)，为什么不把 E 放进去？

因为：
    
        FIRST 集只看第一个可能出现的终结符

对于 

    F→(E)

推导时：

    F ⇒ (E)

无论 E 最终推导成什么：

    (id)
    (id+id)
    ((id))

最左边最先出现的永远都是：(

所以只加入：(

而不是加入 E 的 FIRST 集。

举个具体例子

假设：

    E⇒id+id

则：

    F ⇒ (E)
      ⇒ (id+id)

整个串的第一个终结符是：(

不是：id

因为括号已经排在最前面了。

再举一个：

如果文法是：

    A→BC

那么：如果 B 不能推出 ε，

则：

    FIRST(A)=FIRST(B)

因为最先出现的一定是 B 推导出的终结符。

总结

对于：

    F→(E) ∣ id
    候选式 (E) 的第一个终结符是 (
    候选式 id 的第一个终结符是 id

因此：

    FIRST(F)={(, id}
	​
记忆一句话：

    FIRST 集只关心“最左边第一个可能出现的终结符”。
